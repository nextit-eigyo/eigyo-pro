# 地域ニュースアンテナ 自動収集ルーチン 仕様書

このファイルは「平日朝の自動収集（クラウド定期実行）」が毎回従う指示書です。
出力先：`data/region-news.json`（このスキーマを厳守）。

## 実行頻度
- 平日（月〜金）朝。土日祝はスキップしてよい。

## やること（毎回）
1. まず `git pull`（最新化）。
2. 下記5エリアそれぞれについて、`web_search` で**当日〜直近の薬局関連ニュース**を収集する。
   - テーマ：M&A／事業承継／大手再編・出店／調剤報酬改定の影響／新規開局・クリニック開業。
   - 検索語の例：「<県名> 調剤薬局 M&A OR 事業承継 OR 開局 2026」をエリア内の主要県で。
3. 各ニュースを下記スキーマの item にする。
   - **会社名・店舗数・件数・金額・時期** を含むものは必ず `needsVerify: true`。
   - `pref` は対象の県（複数なら「熊本・宮崎」のように・区切り、エリア全体は「九州全域」等）。
   - `date` は記事のおおよその日付（不明なら収集日 generatedAt）。
   - 出典 `source.url` は必ず実在URL。出典が無い・裏が取れないものは載せない。
4. 各エリアの `coverage`（good/medium/thin）と `coverageNote` を実態に合わせて更新。
5. **新規開局のピンポイント情報はネットに出にくい**ため、無理に推測で作らない。拾えなければ業界動向＋一次情報源（県薬剤師会等）でよい。
6. `generatedAt` を実行日(YYYY-MM-DD)に更新。各 branch は前回分を土台に、**新着を上に**追加し、古すぎ（目安30日超）や重複は整理。
7. `data/region-news.json` を上書き保存 → `git add` → コミット（メッセージ例：`chore(news): 地域ニュース自動更新 YYYY-MM-DD`）→ `git push`。
   - 触るのは `data/region-news.json` のみ。**index.html や他データには一切触れない**（多PC同期事故防止）。

## エリア定義（拠点＝担当リージョン）
- 神戸本社／関西：兵庫・大阪・京都・滋賀・奈良・和歌山
- 福岡営業所／九州：福岡・佐賀・長崎・熊本・大分・宮崎・鹿児島
- 高知営業所／四国：高知・徳島・愛媛・香川
- 東京事業所／関東：東京・神奈川・埼玉・千葉・茨城・栃木・群馬
- 新潟営業所／甲信越・北陸：新潟・長野・富山・石川・福井・山梨

## 出力スキーマ（region-news.json）
```json
{
  "generatedAt": "YYYY-MM-DD",
  "branches": [
    {
      "name": "福岡営業所",
      "region": "九州エリア",
      "prefectures": ["福岡","佐賀","長崎","熊本","大分","宮崎","鹿児島"],
      "coverage": "good|medium|thin",
      "coverageNote": "一言",
      "items": [
        {
          "pref": "熊本",
          "date": "YYYY-MM-DD",
          "type": "M&A | 承継 | 大手再編 | 改定・業界動向 | 業界動向 | 一次情報源 | ターゲット把握",
          "needsVerify": true,
          "title": "見出し",
          "summary": "1〜2文。営業目線(なぜ卸訪問/開局/入替に効くか)。",
          "source": { "label": "媒体名", "url": "https://..." }
        }
      ]
    }
  ]
}
```

## 守ること
- 数字・固有名詞は誇張・創作しない。怪しい時は `needsVerify:true`＋本文に「※要確認」。
- 集計・他ファイルには影響させない。区切りの良いところで終え、失敗時は何もコミットしない。
